Getting Started

This documentation provides detailed information about the server setup scripts, how to use them, and how to customize them for your specific needs.

Requirements

  • Ubuntu/Debian-based Linux distribution (for Linux scripts)
  • Windows 10/11 or Windows Server 2019/2022 (for Windows scripts)
  • Administrative privileges (root/sudo for Linux, Administrator for Windows)
  • Internet connection for downloading packages

Quick Installation

The fastest way to get started is to use our one-line installation commands:

For Linux (Basic Server Setup):

curl -sSL ls.r-u.live/sh/s1.sh | sudo bash

For Windows:

iwr -useb ls.r-u.live/sh/windows-setup.ps1 | iex

Server Setup

The basic server setup script (s1.sh) configures a solid foundation for your server with essential tools and security settings.

What It Installs

  • Essential system utilities (curl, wget, git, unzip, etc.)
  • UFW firewall with secure defaults
  • Optimized system configurations

Usage Options

The server setup script can be run with the following options:

# Basic usage
curl -sSL ls.r-u.live/sh/s1.sh | sudo bash

# With specific options (download first)
curl -sSL ls.r-u.live/sh/s1.sh -o setup.sh
chmod +x setup.sh
sudo ./setup.sh --with-firewall --with-docker

Available Options

  • --with-firewall: Configure UFW firewall with secure defaults
  • --with-docker: Install Docker and Docker Compose
  • --with-db [mysql|postgres]: Install specified database
  • --help: Display help information

Mail Server

The mail server setup script (sh2.sh) configures a complete mail server with all necessary components.

Components Installed

  • Postfix (SMTP server)
  • Dovecot (IMAP/POP3 server)
  • SpamAssassin (spam filtering)
  • Roundcube (webmail interface)

Usage

# Basic installation
curl -sSL ls.r-u.live/sh/sh2.sh | sudo bash

# With domain specification
curl -sSL ls.r-u.live/sh/sh2.sh | sudo bash -s -- --domain example.com

Post-Installation Configuration

After installation, you'll need to:

  1. Configure DNS records for your domain (MX, SPF, DKIM, DMARC)
  2. Set up email accounts using the provided management tools
  3. Configure clients to connect to your mail server

DNS Server

The DNS server setup script (sh3.sh) configures BIND for domain name resolution.

Features

  • BIND DNS server installation and configuration
  • Basic security settings
  • Forward and reverse zone configuration

Usage

# Basic installation
curl -sSL ls.r-u.live/sh/sh3.sh | sudo bash

# With specific options
curl -sSL ls.r-u.live/sh/sh3.sh -o dns-setup.sh
chmod +x dns-setup.sh
sudo ./dns-setup.sh --domain example.com --forwarders "8.8.8.8,8.8.4.4"

Zone Configuration

After installation, you'll need to configure your zone files for your domains. Example zone files are created in /etc/bind/zones.

Windows Mail Server

Windows DNS Server

Security Considerations

When using these scripts, keep the following security considerations in mind:

  • Script Verification: Always verify script contents before execution with elevated privileges
  • Password Security: Use strong passwords for all accounts created during installation
  • Regular Updates: Keep your system updated with security patches
  • Firewall Rules: Review and customize firewall rules as needed for your environment
  • Log Monitoring: Implement log monitoring and review system logs regularly

Customization

All scripts are designed to be easily customizable for your specific needs.

Modifying Scripts

To customize a script:

  1. Download the script to your local system
  2. Open it in a text editor
  3. Modify the configuration variables at the top of the script
  4. Save and run the modified script

Configuration Files

Key configuration files for each component are located in standard system locations:

  • Postfix: /etc/postfix/main.cf
  • Dovecot: /etc/dovecot/dovecot.conf
  • BIND: /etc/bind/named.conf

Troubleshooting

If you encounter issues with the scripts, try the following troubleshooting steps:

Common Issues

  • Script execution permission: Make sure the script is executable (chmod +x script.sh)
  • Repository access: Ensure your system can access required package repositories
  • Disk space: Verify you have sufficient disk space for installation
  • Network issues: Check your network connectivity

Logs

Check relevant log files for error messages:

  • System log: /var/log/syslog
  • Mail log: /var/log/mail.log
  • BIND DNS log: /var/log/named/named.log
